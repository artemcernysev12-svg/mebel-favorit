ВАЖНАЯ НАХОДКА — БАГ ID КАРТОЧЕК МИКОН
---------------------------------------
При сверке связки с каталогом обнаружено, что 22 карточки
Микон в catalog.js имели УКОРОЧЕННЫЕ Авито-ID — где-то по
дороге первая цифра (а в двух случаях и последняя) была
обрезана. Скорее всего, при импорте из Excel: числа теряли
ведущий 0 или старший разряд.

Примеры исправлений:
  Было в catalog.js     Стало (правильный Авито-ID)   Что это
  ────────────────────  ──────────────────────────    ───────
  059052075             8059052075                    Спальня Асти
  059327421             8059327421                    Детская Лючия
  31612429              4316124290                    Стенка Сидней 2
  027288024             8027288024                    Прихожая Витра
  ... ещё 18 таких

Всего исправлено: 22 карточки.

Что это значило для сайта:
  ❌ До правки: при открытии этих карточек блок "Состав комплекта"
     был пуст, потому что код искал __KITCHEN_COMP__["059052075"],
     а в файле связки лежит под "8059052075".

  ✅ После правки: связка ложится идеально, состав отображается.

Безопасность правки:
  • Конфликтов с другими ID нет (все 22 новых ID уникальны).
  • Поле 'avito' в карточке трогать не пришлось — там уже
    правильная ссылка.
  • Поле 'mu' (multi-listing group) — старых ID не содержит.
  • Sitemap.xml перегенерирован с новыми ID.

Ещё одна правка: в свежем файле связки была опечатка
  8059033379 → правильно 8059033378
По твоему уточнению — поправили в выходных данных.
