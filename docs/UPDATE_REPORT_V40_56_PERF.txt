ПОЧЕМУ ЭТО БЕЗОПАСНО
====================

Раньше эти три файла (~346 КБ суммарно) грузились синхронно: браузер
останавливал парсер HTML, скачивал и выполнял каждый по очереди, и
только потом продолжал отрисовку страницы.

Теперь с defer они скачиваются параллельно с разбором HTML, а
выполняются после него — но всё ещё ДО события DOMContentLoaded.

Все функции, которые читают данные из этих скриптов, уже имеют
проверку наличия глобала:

    function getWardrobeAtticInfo(it){
      if(!it || !window.__WARDROBE_ATTIC__) return null;
      ...
    }
    function getBedOptions(it){
      if(!it || !window.__BED_OPTIONS__) return null;
      ...
    }
    function getWardrobeCoupeInfo(it){
      if(!it || !window.__WARDROBE_COUPE__) return null;
      ...
    }

Эти функции вызываются только при открытии карточки товара (по клику
пользователя), а в этот момент defer-скрипты уже давно выполнены.

Никаких top-level вызовов, которые бы зависели от этих глобалов в
момент парсинга, в коде нет — проверено по grep всех использований
WARDROBE_ATTIC / BED_OPTIONS / WARDROBE_COUPE в index.html.


ОЖИДАЕМЫЙ ЭФФЕКТ
================

- На мобильном 4G/3G и медленном вайфае: первый экран рисуется
  заметно раньше — ~346 КБ перестают блокировать парсер.
- На быстром Wi-Fi разница на глаз почти незаметна, но в Lighthouse
  и PageSpeed Insights должно улучшиться TBT (Total Blocking Time)
  и FCP (First Contentful Paint).
- Функциональных изменений ноль: модалки шкафов, кроватей,
  купе, кухонь, фильтры, конструктор, остатки, админка работают
  точно так же.


КАК ПРОВЕРИТЬ ПОСЛЕ ДЕПЛОЯ
==========================

1) Открыть https://мебель-фаворит.рф/ в режиме инкогнито (чтобы
   обойти кэш).
2) F12 → вкладка Network → отфильтровать по JS.
3) Найти строки wardrobe-attic.js, bed-options.js, wardrobe-coupe.js.
   У всех должна быть колонка Priority = Low (или High без
   "blocking") — раньше там был синхронный blocking.
4) Открыть карточку любого шкафа Велес или Альте — блок выбора
   "Только шкаф / Шкаф + антресоль" должен появиться и работать.
5) Открыть карточку шкафа-купе (например Гранд или Элегант) —
   блок "Итог комплектации шкафа-купе" должен показывать корпус
   и фасады.
6) Открыть карточку любой кровати Велес — варианты основания и
   доп. опции должны быть на месте.

Если что-то из пунктов 4–6 не работает после обновления, значит
файл wardrobe-* не успел загрузиться к моменту клика. На практике
этого не произойдёт (defer-скрипты грузятся в первые 1–2 секунды),
но если случится — достаточно подождать секунду и кликнуть ещё раз.


ЧТО НЕ ТРОГАЛОСЬ
================

- Логика каталога, фильтров, конструктора кухонь, остатков, админки
- Service worker (по-прежнему self-uninstall)
- Стили (CSS), шаблоны карточек, модальные окна
- Yandex Metrika, Google Search Console, schema.org разметка
- robots.txt, sitemap.xml, CNAME
- catalog.js (1.6 МБ) — разбивать пока не стали, риск выше выгоды


ЧТО МОЖНО СДЕЛАТЬ В СЛЕДУЮЩИХ ИТЕРАЦИЯХ
========================================

- Разбить catalog.js по категориям (диваны, шкафы, кухни и т.д.)
  и подгружать только нужные при выборе фильтра. Это самый
  большой потенциальный выигрыш, но требует переписать секцию
  загрузки и фильтрации.
- Включить полноценный service worker с кэшированием шрифтов,
  CSS и каталога. Сейчас sw.js — заглушка, выгружающая саму себя.
- Сжать og-preview.jpg (64 КБ) до ~25–30 КБ через mozjpeg или
  squoosh без видимой потери качества.
- Перевести шрифты Unbounded и Golos Text на самохост (положить
  woff2 в assets/) и убрать запрос к fonts.googleapis.com — это
  даст ещё ~100–200 мс на медленных сетях.
